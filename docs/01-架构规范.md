## 1. 项目概述

**项目定位**:内部多账号防关联浏览器 + AI 操作执行环境,**Electron 桌面客户端 + 中央 Node.js 服务器**架构。

**两类用户**:
- **人类用户**(10 人团队):员工日常多账号操作韩国店铺
- **AI 用户**(秘书 AI / Codex):通过双轨接口代员工执行任务

**两层数据**:
- **服务器**:账号密码、操作日志、权限、AI 任务、代理 IP 配置
- **客户端**:UI 缓存、本地 session 数据(cookie 等加密文件)

## 2. 技术栈

| 层 | 技术 | 版本 | 选型理由 |
|---|---|---|---|
| 客户端 UI | React + TypeScript + Vite | React 19,TS 5.x | 标准组合,Codex 熟 |

| 指纹注入 | 自研 stealth 脚本(参考 puppeteer-extra-plugin-stealth 算法) | - | 紫鸟核心也是这套思路 |

## 5. 数据库表设计(PostgreSQL)

```sql
-- 用户(包括人和 AI)
CREATE TABLE users (
  id SERIAL PRIMARY KEY,
  username VARCHAR(64) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,         -- bcrypt cost=12
  role VARCHAR(20) NOT NULL,                    -- 'boss' | 'manager' | 'staff' | 'ai'
  team_id INT,
  display_name VARCHAR(64),
  status VARCHAR(20) DEFAULT 'active',          -- 'active' | 'disabled' | 'frozen'
  frozen_until TIMESTAMP,                       -- AI 触发红灯被冻结到此时间
  created_at TIMESTAMP DEFAULT NOW(),
  last_login_at TIMESTAMP
);

-- 团队
CREATE TABLE teams (
  id SERIAL PRIMARY KEY,
  name VARCHAR(64) NOT NULL,
  manager_id INT REFERENCES users(id)
);

-- 店铺
CREATE TABLE shops (
  id SERIAL PRIMARY KEY,
  name VARCHAR(128) NOT NULL,
  platform VARCHAR(32) NOT NULL,                -- 'naver_smartstore' | 'coupang' | 'gmarket' | '11st'
  team_id INT REFERENCES teams(id),
  default_url VARCHAR(255),
  proxy_id INT REFERENCES proxies(id),
  fingerprint_config JSONB,                     -- 首次创建固化的指纹随机种子
  status VARCHAR(20) DEFAULT 'active',
  created_at TIMESTAMP DEFAULT NOW()
);

-- 店铺账号密码(独立表,权限严格)
CREATE TABLE shop_credentials (
  id SERIAL PRIMARY KEY,
  shop_id INT UNIQUE REFERENCES shops(id) ON DELETE CASCADE,
  username VARCHAR(255) NOT NULL,               -- 平台账号(明文,不敏感)
  password_encrypted BYTEA NOT NULL,            -- AES-256-GCM
  password_iv BYTEA NOT NULL,
  password_tag BYTEA NOT NULL,
  updated_at TIMESTAMP DEFAULT NOW(),
  updated_by INT REFERENCES users(id)
);

-- 员工 → 店铺(多对多)
CREATE TABLE shop_assignments (
  user_id INT REFERENCES users(id),
  shop_id INT REFERENCES shops(id),
  granted_at TIMESTAMP DEFAULT NOW(),
  granted_by INT REFERENCES users(id),
  PRIMARY KEY (user_id, shop_id)
);

-- 操作日志(只允许 INSERT,触发器禁 UPDATE/DELETE)
CREATE TABLE action_logs (
  id BIGSERIAL PRIMARY KEY,
  actor_id INT REFERENCES users(id),
  actor_type VARCHAR(10),                       -- 'human' | 'ai'
  shop_id INT REFERENCES shops(id),
  action_type VARCHAR(64) NOT NULL,
  action_payload JSONB,
  before_value JSONB,
  after_value JSONB,
  risk_level VARCHAR(10),                       -- 'green' | 'yellow' | 'red'
  approval_status VARCHAR(20),                  -- 'auto' | 'pending' | 'approved' | 'denied'
  approved_by INT REFERENCES users(id),
  ip_address VARCHAR(64),
  user_agent TEXT,
  screenshot_path VARCHAR(255),
  created_at TIMESTAMP DEFAULT NOW(),
  retention_until TIMESTAMP                     -- created_at + 180 天
);
CREATE INDEX idx_action_logs_actor ON action_logs(actor_id, created_at DESC);
CREATE INDEX idx_action_logs_shop ON action_logs(shop_id, created_at DESC);
CREATE INDEX idx_action_logs_risk ON action_logs(risk_level, approval_status);

-- AI 任务
CREATE TABLE ai_tasks (
  id SERIAL PRIMARY KEY,
  ai_id INT REFERENCES users(id),
  shop_id INT REFERENCES shops(id),
  command VARCHAR(64) NOT NULL,
  payload JSONB,
  status VARCHAR(20) DEFAULT 'pending',         -- pending|approved|denied|executing|done|failed
  risk_level VARCHAR(10),
  approval_required BOOLEAN,
  assigned_to INT REFERENCES users(id),
  result JSONB,
  created_at TIMESTAMP DEFAULT NOW(),
  approved_at TIMESTAMP,
  executed_at TIMESTAMP
);

-- 代理池
CREATE TABLE proxies (
  id SERIAL PRIMARY KEY,
  provider VARCHAR(32),                         -- 'lunaproxy' | '922s5'
  protocol VARCHAR(10),                         -- 'http' | 'socks5'
  host VARCHAR(128),
  port INT,
  username VARCHAR(64),
  password_encrypted BYTEA,
  country VARCHAR(8) DEFAULT 'KR',
  city VARCHAR(64),
  status VARCHAR(20) DEFAULT 'active',          -- 'active' | 'broken' | 'reserved'
  last_check_at TIMESTAMP,
  bound_shop_id INT UNIQUE REFERENCES shops(id) -- 一店一 IP 固定
);

-- 应急事件
CREATE TABLE emergency_events (
  id SERIAL PRIMARY KEY,
  triggered_by INT REFERENCES users(id),
  event_type VARCHAR(32),                       -- 'lockout' | 'boundary_red_attempted' | ...
  scope VARCHAR(20),                            -- 'all' | 'team' | 'employee' | 'shop'
  scope_target_id INT,
  reason TEXT,
  affected_users INT[],
  triggered_at TIMESTAMP DEFAULT NOW()
);
```

**数据库触发器**:

```sql
CREATE OR REPLACE FUNCTION prevent_action_log_modify() RETURNS TRIGGER AS $$
BEGIN
  RAISE EXCEPTION '操作日志不可修改/删除';
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER no_update_action_logs BEFORE UPDATE ON action_logs
FOR EACH ROW EXECUTE FUNCTION prevent_action_log_modify();

CREATE TRIGGER no_delete_action_logs BEFORE DELETE ON action_logs
FOR EACH ROW EXECUTE FUNCTION prevent_action_log_modify();
```

(180 天过期清理走另一条管理员通道,不通过普通 DELETE)

## 7. 防关联实现(中层指纹隔离)

### 7.1 浅层(Electron 原生)

每店铺独立 `session.fromPartition('persist:shop-{shopId}')`:
- ✅ Cookie 隔离
- ✅ LocalStorage 隔离
- ✅ IndexedDB 隔离
- ✅ Service Worker 隔离
- ✅ 缓存隔离

### 7.2 中层(自研 stealth 脚本注入)

每店铺首次创建时,**随机生成并固化**一份指纹配置(存 `shops.fingerprint_config`),例如:

```json
{
  "userAgent": "Mozilla/5.0 ... Chrome/121.0.0.0 Safari/537.36",
  "platform": "Win32",
  "language": "ko-KR",
  "languages": ["ko-KR", "ko", "en-US"],
  "canvasSeed": 12345,
  "webglVendor": "Google Inc. (NVIDIA)",
  "webglRenderer": "ANGLE (NVIDIA, NVIDIA GeForce GTX 1660 ...)",
  "audioOffset": 0.000123,
  "hardwareConcurrency": 8,
  "screenWidth": 1920,
  "screenHeight": 1080,
  "timezone": "Asia/Seoul",
  "fonts": ["Malgun Gothic", "Nanum Gothic", "..."]
}
```

**关键原则:固化后绝不变化**(变化才是关联破绽)。

启动每个 BrowserView 前,通过 `webContents.executeJavaScript(stealthScript, true)` 在 `dom-ready` 时机注入(伪代码):

```javascript
// 1. UA / 平台 / 语言
Object.defineProperty(navigator, 'userAgent', { get: () => CFG.userAgent });
Object.defineProperty(navigator, 'platform', { get: () => CFG.platform });
Object.defineProperty(navigator, 'language', { get: () => CFG.language });
Object.defineProperty(navigator, 'languages', { get: () => CFG.languages });
Object.defineProperty(navigator, 'hardwareConcurrency', { get: () => CFG.hardwareConcurrency });

// 2. Canvas 指纹(基于 seed 加微小噪声,确保每店铺哈希不同)
const origToDataURL = HTMLCanvasElement.prototype.toDataURL;
HTMLCanvasElement.prototype.toDataURL = function(...args) {
  const ctx = this.getContext('2d');
  if (ctx) {
    const imgData = ctx.getImageData(0, 0, this.width, this.height);
    for (let i = 0; i < imgData.data.length; i += 4) {
      imgData.data[i] = Math.min(255, imgData.data[i] + (CFG.canvasSeed % 3 - 1));
    }
    ctx.putImageData(imgData, 0, 0);
  }
  return origToDataURL.apply(this, args);
};

// 3. WebGL 指纹(覆盖 UNMASKED_VENDOR_WEBGL/UNMASKED_RENDERER_WEBGL)
const origGetParameter = WebGLRenderingContext.prototype.getParameter;
WebGLRenderingContext.prototype.getParameter = function(param) {
  if (param === 37445) return CFG.webglVendor;
  if (param === 37446) return CFG.webglRenderer;
  return origGetParameter.call(this, param);
};

// 4. AudioContext 指纹
// 5. 时区(覆盖 Intl.DateTimeFormat)
// 6. 字体探测(覆盖 document.fonts.check)
// 7. 屏幕尺寸(screen.width/height)
// 8. WebRTC 关闭(防真实 IP 泄露)
```

## 8. AI 接入双轨

### 8.1 第 1 轨:指令清单(80% 场景)

AI 通过 HTTP API 提交结构化任务:

```http
POST /ai/task
Authorization: Bearer <AI 长期 token>

{
  "command": "product.price.update",
  "shop_id": 3,
  "payload": {
    "sku_id": "ABC-123",
    "new_price": 8500,
    "reason": "竞品降价 8800,跟价"
  }
}
```

服务器流程:
1. 边界系统判定 risk_level
2. green → 直接派发
3. yellow → 等审批
4. red → 拒绝 + 警示

支持的 command 列表见 `02-边界系统清单.md`。

### 8.2 第 2 轨:上帝视角 CDP(20% 场景)

AI 通过 Chrome DevTools Protocol 直接控制 BrowserView:

- Electron 启动时开放 `--remote-debugging-port=9222`(只监听 127.0.0.1)
- 服务器签发短期 CDP token(5 分钟,绑定具体 shop_id)
- AI 用 token 连接 `wss://browser-api.xinhuonianhua.com/cdp/{token}`
- 服务器把 CDP 命令中转到员工客户端的 9222 端口

**所有 CDP 操作仍走中间代理审计**:
```
AI ─→ 服务器 CDP 代理 ─→ Electron CDP ─→ BrowserView
       ↓
    每个 CDP 命令记一条 action_logs
```

避免"AI 直连 = 没审计"的漏洞。CDP 期间触发红灯指令直接断连。

## 9. 边界系统执行引擎

详见 [02-边界系统清单.md](02-边界系统清单.md)。

核心逻辑:
1. AI 提交 → 服务器查动作清单 → 计算 risk_level
2. green → 自动执行
3. yellow → 暂存,WebSocket 通知主管,等审批
4. red → 拒绝 + 写 emergency_events + 钉钉/微信告警老板 + AI 账号冻结 1 小时

### 11.3 主密钥管理(最敏感)

**核心问题:AES-256 主密钥放哪?**

方案:
- 主密钥(32 字节)以文件形式保存在**老板电脑**:`C:\Users\tianc\.pandao-key\master.key`
- 服务器启动时**不持有**主密钥(防止服务器被拖库泄露)
- 老板每天上班启动服务器后,手动 unlock:
  ```powershell
  curl -X POST https://browser-api.xinhuonianhua.com/admin/unlock `
       -H "Authorization: Bearer <老板 JWT>" `
       --data-binary "@C:\Users\tianc\.pandao-key\master.key"
  ```
- 服务器内存里持有主密钥,所有加解密在内存内完成
- 服务器重启后必须老板重新 unlock,否则 API 不能解密(报 503)

**保险**:老板出差时设一个备用解锁人(主管),通过钉钉一次性传 master.key 文件。

## 12. 安全措施

| 项 | 实现 |
|---|---|
| 传输加密 | HTTPS(TLS 1.2+) |
| 密码存储 | AES-256-GCM,主密钥不入服务器磁盘 |
| 用户密码哈希 | bcrypt(cost=12) |
| JWT | 短期(2 小时) + refresh token(7 天) |
| 短期解密 token | 60 秒有效,只能用一次 |
| 防重放 | nonce + timestamp |
| SQL 注入防护 | 参数化查询(pg 库 + 严格类型) |
| XSS 防护 | React 默认转义 + 严格 CSP |
| 客户端启动认证 | 账号密码登录 |
| 操作日志不可改 | 数据库触发器禁 UPDATE/DELETE |
| 主密钥泄露应急 | 旋转主密钥流程(后续补 docs/security-keyrotation.md) |
| CDP 端口 | 仅监听 127.0.0.1,服务器代理转发 |
| WebSocket 鉴权 | JWT,断开后重连需重新鉴权 |
