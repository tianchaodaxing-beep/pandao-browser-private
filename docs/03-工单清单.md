### WO-003:店铺账号库 + AES-256-GCM 加密

| 项 | 内容 |
|---|---|
| 优先级 | P0 |
| 依赖 | WO-002 |
| 预估 | 3-4 天 |

**任务**:
1. 创建 `shops` / `shop_credentials` / `shop_assignments` 表
2. 服务器实现:
   - `POST /shops` 老板创建店铺
   - `POST /shops/:id/credentials` 设置账号密码(立即 AES-256-GCM 加密入库)
   - `POST /shops/:id/assign` 分配店铺给员工
   - `GET /shops` 列出当前用户能访问的店铺(按 RBAC 过滤)
3. 主密钥管理(关键):
   - 服务器启动**不持有**主密钥
   - 提供 `POST /admin/unlock` 接口,老板上传 master.key 后注入服务器内存
   - 服务器内存中保存主密钥,所有 AES 加解密在内存完成
   - 服务器重启后必须老板手动 unlock,否则 API 返回 503
4. CLI 工具:`npm run keygen` 生成新的 master.key(老板首次部署用)
5. CLI 工具:`npm run rotate-key` 旋转主密钥(后续 P2 阶段用,P0 先留接口)

**验收**:
```powershell
# 1. 生成主密钥
cd F:\pandao-browser\server
npm run keygen
# 输出 master.key 到 C:\Users\tianc\.pandao-key\master.key

# 2. 启动服务器,确认锁定
npm run dev
curl http://127.0.0.1:3001/admin/lock-status
# 期望 {"locked":true}

# 3. 解锁
$bossToken = "<老板登录后的 JWT>"
curl -X POST http://127.0.0.1:3001/admin/unlock `
     -H "Authorization: Bearer $bossToken" `
     --data-binary "@C:\Users\tianc\.pandao-key\master.key"

# 4. 创建店铺并设密码
curl -X POST http://127.0.0.1:3001/shops `
     -H "Authorization: Bearer $bossToken" `
     -d '{\"name\":\"测试店铺\",\"platform\":\"naver_smartstore\"}'
curl -X POST http://127.0.0.1:3001/shops/1/credentials `
     -H "Authorization: Bearer $bossToken" `
     -d '{\"username\":\"test@example.com\",\"password\":\"abc123\"}'

# 5. 验证密码已加密
psql -d pandao_browser -c "SELECT password_encrypted FROM shop_credentials WHERE shop_id=1;"
# 期望:bytea 字段是乱码(不是 'abc123')
```
