# npm audit 例外说明

更新时间：2026-05-12 14:55 Asia/Shanghai

## 保留原因

WO-002 已锁定服务端保持 Fastify `^4.29.1`，禁止升级到 Fastify 5.x。

当前 `npm audit --omit=dev` 的自动修复路径会引入以下破坏性升级：

- `fastify` / `fast-uri`：需要升级 Fastify 5.x。
- `@fastify/jwt` / `fast-jwt`：需要升级 `@fastify/jwt@10`，该版本启动时报错 `expected '5.x' fastify version`，与 Fastify 4 决策冲突。

## 当前处理

- 保持 `fastify@^4.29.1`，不升级到 5.x。
- 保持 `@fastify/jwt@8.0.1`，这是 Fastify 4 可运行版本。
- `server/package.json` 和根 `package.json` 已记录 overrides 意图：`fast-uri@3.1.2`、`fast-jwt@6.2.4`。npm workspace 对 Fastify 4 子依赖仍会保留审计警告，进入生产前需由主控统一决定安全补丁或 Fastify 5 升级窗口。
- 服务端启动时强制要求 `JWT_SECRET` / `JWT_REFRESH_SECRET`，不允许空密钥或默认密钥；签名算法固定 HS256。
- WO-008 引入 `@fastify/multipart@8.3.1` 与 Fastify 4 兼容，`npm audit --omit=dev -w server` 未新增 multipart 相关漏洞；当前警告仍来自 Fastify 4 / fast-uri / @fastify/jwt / fast-jwt 这条已锁定链路。

## 例外数量

- `npm audit --omit=dev` 例外包族：2 个，`fast-jwt`、`fast-uri/Fastify 4`。
- 2026-05-12 实测 audit vulnerability 记录数：7 条；不通过升级 Fastify 5 / @fastify/jwt 10 自动修复。
